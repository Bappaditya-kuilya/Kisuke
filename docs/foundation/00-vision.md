# Vision

> Source: MASTER_SPECIFICATION.md §1

---

## Purpose

Kisuke exists to reduce the time and mental effort required to resume meaningful work.

It reconstructs the working context that existed before an interruption instead of requiring manual reconstruction from notes, memory, or scattered artifacts.

---

## Problem

Knowledge is rarely lost.

Context is.

Modern tools store information but do not reconstruct the state of understanding required to continue work.

Every interruption forces manual context reconstruction.

This repeated reconstruction is the primary problem Kisuke solves.

---

## Vision

Kisuke is a local-first context reconstruction system.

Given any point in previous work, Kisuke reconstructs the working context necessary to continue that work with minimal cognitive effort.

---

## Goals

- Reduce context switching cost.
- Resume work in under two minutes.
- Reconstruct context instead of displaying notes.
- Operate offline for core functionality.
- Integrate with existing tools.
- Preserve architectural simplicity.
- Maintain a single source of truth.

---

## Non-Goals

Kisuke is not:

- A note-taking application.
- A search-first system.
- A cloud platform.
- An autonomous AI agent.
- A replacement for Git.
- A replacement for Obsidian.
- A replacement for Google Calendar.

---

## Core Promise

> Kisuke does not return notes.

> Kisuke returns working context.

---

## Success Criteria

Kisuke succeeds when it:

- Reduces context reconstruction time.
- Minimizes manual searching.
- Reconstructs accurate working context.
- Preserves architectural consistency.

---

## Product Philosophy

- Local-first.
- Markdown-first.
- Git-native.
- Offline-first.
- Resume before search.
- Reference over duplication.
- Integrate before rebuild.
- AI is optional.
