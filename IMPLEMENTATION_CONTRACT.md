# IMPLEMENTATION CONTRACT

## Purpose

This repository follows Documentation First Development.

Implementation follows documentation.

Documentation never follows implementation.

---

# Workflow

Idea

↓

RFC (optional)

↓

Documentation

↓

Review

↓

Implementation

↓

Tests

↓

Merge

---

# Rules

* Do not redesign the architecture.
* Do not add new features.
* Do not modify the domain model.
* Do not introduce hidden behavior.
* Do not duplicate ownership.
* Do not bypass documentation.

---

# Definition of Done

Every milestone must include:

* Working implementation
* Tests
* Documentation alignment
* Passing review

---

# Architecture Review

If implementation reveals a design flaw:

* Stop implementation.
* Create an RFC.
* Wait for approval.

Never silently change architecture.
