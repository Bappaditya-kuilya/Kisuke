# ADR-002

# Title

Markdown as the Canonical Data Store

---

## Status

Accepted

---

## Context

Kisuke needs a storage format that is:

- Human-readable
- Git-friendly
- Long-lived
- Portable
- AI-friendly
- Tool-independent

---

## Decision

Markdown is the canonical storage format.

Everything else is derived.

Derived artifacts include:

- SQLite
- Search Index
- Cache
- Embeddings
- AI Outputs

Derived artifacts may be deleted and rebuilt at any time.

---

## Consequences

### Positive

- Human editable
- Git versioned
- Portable
- Future proof
- Easy backup

### Negative

- Parsing complexity
- Slightly slower than databases

---

## Alternatives Considered

SQLite as canonical.

Rejected.

Reason:

Not human-readable.

---

JSON as canonical.

Rejected.

Reason:

Poor long-term editing experience.
