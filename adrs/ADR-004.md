# ADR-004

# Title

Plugin Architecture

---

## Status

Accepted

---

## Context

Kisuke must remain small, stable and maintainable while allowing future extensions.

---

## Decision

All optional functionality is implemented as plugins.

The Core exposes stable interfaces.

Plugins interact only through public APIs.

Plugins may never modify Core directly.

---

## Consequences

### Positive

- Small core
- Easy extensibility
- Safe experimentation
- Independent plugins

### Negative

- More interface design

---

## Alternatives Considered

Everything in Core.

Rejected.

Reason:

Creates an ever-growing monolith.
