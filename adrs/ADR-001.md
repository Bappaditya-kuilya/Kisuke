# ADR-001

# Title

Architecture Principles

---

## Status

Accepted

---

## Context

Kisuke is a local-first context reconstruction engine.

Its architecture must remain stable for years while allowing integrations, plugins and multiple user interfaces.

Without clear architectural constraints, implementation drift will occur.

---

## Decision

Kisuke adopts the following principles:

- Clean Architecture
- Local-first
- Markdown-first
- Offline-first core
- Git-native history
- Provider-independent AI
- Documentation-first development
- Single ownership
- Reference over duplication
- Resume before search
- Integrate before rebuilding

The Domain layer is independent of:

- UI
- Database
- AI providers
- External services

User interfaces are adapters.

The CLI is the reference implementation.

Future interfaces (Desktop, Web, Mobile) consume the same Application layer.

---

## Consequences

### Positive

- Stable architecture
- Easy testing
- Replaceable integrations
- Multiple frontends
- Long-term maintainability

### Negative

- Slightly more initial engineering effort
- Strict architectural boundaries

---

## Alternatives Considered

### Monolith

Rejected.

Reason:

Too tightly coupled.

---

### MVC

Rejected.

Reason:

Business logic leaks into presentation.

---

### Framework-first

Rejected.

Reason:

Creates vendor lock-in.
